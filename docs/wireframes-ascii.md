# IITMS ASCII Wireframes (Full Page Details + Component Consistency)

This file defines consistent ASCII wireframes for every page route and the exact component appearance pattern to use.

## 1) Global Layout Contract (All App Pages)

+--------------------------------------------------------------------------------------------------+
| SIDEBAR (fixed 264px)  | TOP BAR (64px): [Page Title] [Search?] [Alerts] [Avatar Menu]         |
|------------------------+--------------------------------------------------------------------------|
| Nav Group A            | CONTENT AREA (max-w-7xl, centered, px-6, py-6, vertical gap = 24px)     |
| Nav Group B            |                                                                          |

## 2) Component Blueprints (Visual/Structure Consistency)

### 2.1 PageHeader

```text
+--------------------------------------------------------------------------------+
| Title (text-3xl semibold)                                 [Primary] [Secondary]|
| Description (text-sm muted)                                                   |
+--------------------------------------------------------------------------------+
```

Rules:
- Left aligned title + description.
- Action buttons right aligned on desktop, stacked below title on mobile.
- Always first block in content area.

### 2.2 Section Card

```text
+--------------------------------------------------------------------------------+
| Section Title (text-xl)                                     [Action] [Action]  |
| Section Description (optional, text-sm muted)                                  |
|--------------------------------------------------------------------------------|
| Section body content                                                           |
+--------------------------------------------------------------------------------+
```

Rules:
- Rounded large radius, `border + shadow-sm`.
- 24px inner padding desktop, 16px mobile.

### 2.3 Stat Card

```text
+-----------------------------+
| Label (text-xs uppercase)   |
| Value (text-2xl bold)       |
| Delta/Hint (text-xs muted)  |
+-----------------------------+
```

Rules:
- 3-up desktop, 1-up mobile.
- Equal heights.

### 2.4 Filter Bar

```text
+--------------------------------------------------------------------------------+
| [Search....................] [Status v] [Type v] [Date v]      [Reset] [Apply] |
+--------------------------------------------------------------------------------+
```

Rules:
- Always appears above table/list.
- Same control order: Search -> categorical filters -> date -> actions.

### 2.5 Data Table

```text
+--------------------------------------------------------------------------------+
| Col A | Col B | Col C | Status | Updated | Actions                             |
|--------------------------------------------------------------------------------|
| ...                                                                          .. |
|--------------------------------------------------------------------------------|
| Pagination: < Prev   1 2 3   Next >                                            |
+--------------------------------------------------------------------------------+
```

Rules:
- Sticky header where practical.
- Row hover and selected states consistent.
- Actions grouped in final column.

### 2.6 Form

```text
+--------------------------------------------------------------------------------+
| Field Label *                                                                   |
| [Input......................................................................]  |
| Help/Error text                                                                 |
|--------------------------------------------------------------------------------|
| Field Label                                                                     |
| [Textarea.................................................................]     |
|--------------------------------------------------------------------------------|
| [Cancel]                                                     [Primary Submit]   |
+--------------------------------------------------------------------------------+
```

Rules:
- Required fields marked `*`.
- Error appears directly under input.
- Submit button pinned at form footer area.

### 2.7 Empty State

```text
+--------------------------------------------------------------------------------+
|                                   (Icon)                                       |
|                            Empty Title (text-lg)                               |
|                      Explanation (text-sm muted, max width)                    |
|                               [Primary Action]                                 |
+--------------------------------------------------------------------------------+
```

Rules:
- Centered content, consistent icon circle treatment.

### 2.8 Detail Panel (for record detail/review)

```text
+------------------------------------+
| Record Title                       |
| Meta line                          |
|------------------------------------|
| Group A fields                     |
|------------------------------------|
| Group B fields                     |
|------------------------------------|
| [Reject]                 [Approve] |
+------------------------------------+
```

Rules:
- Used as right column on desktop, stacked below list on mobile.

### 2.9 Loading Indicator

Full-page loading:

```text
+--------------------------------------------------------------------------------+
|                                                                                |
|                         (spinner icon, 48px)                                   |
|                           Loading...                                           |
|                                                                                |
+--------------------------------------------------------------------------------+
```

Section loading (card skeleton):

```text
+--------------------------------------------------------------------------------+
| Section Title                                                                   |
|--------------------------------------------------------------------------------|
| [====================]                                                          |
| [==========================================]                                   |
| [==============================]                                                |
+--------------------------------------------------------------------------------+
```

Table loading:

```text
+--------------------------------------------------------------------------------+
| Col A | Col B | Col C | Status | Actions                                       |
|--------------------------------------------------------------------------------|
| [-----] [-----------] [------] [-----] [--------]                              |
| [-----] [-----------] [------] [-----] [--------]                              |
| [-----] [-----------] [------] [-----] [--------]                              |
+--------------------------------------------------------------------------------+
```

Rules:
- Full-page loading uses centered spinner + "Loading..." text.
- Section/table loading uses skeletons, not empty blanks.
- Keep layout dimensions stable during loading to avoid layout shift.
- Use the same loading style across all roles and modules.

### 2.10 Error States

Local/section error:

```text
+--------------------------------------------------------------------------------+
| (! icon) Unable to load data                                                   |
| Reason text                                                                     |
| [Try Again]                                                                     |
+--------------------------------------------------------------------------------+
```

Global/page error:

```text
+--------------------------------------------------------------------------------+
|                                                                                |
|                            (warning icon)                                      |
|                         Something went wrong                                   |
|                  An unexpected error occurred                                  |
|                       [Reload] [Return to Home]                                |
|                                                                                |
+--------------------------------------------------------------------------------+
```

Rules:
- Local errors are inline inside the affected section/table/form.
- Global errors are centered full-page surfaces.
- Every async page must define loading, empty, and error states.

## 5) Enforced Consistency Checklist (Use On Every Page)

- [ ] Uses `PageHeader` as first content block.
- [ ] Uses `Section` for grouped content.
- [ ] Uses `EmptyState` for no-data states.
- [ ] Filter order is consistent.
- [ ] Table columns end with `Actions` column.
- [ ] Button hierarchy: one primary action per section.
- [ ] Spacing uses design tokens only.
- [ ] Same status badge styles across modules.
- [ ] Form error/help text placement consistent.
- [ ] Mobile stacking preserves reading order.
- [ ] Loading state exists for full page, section, and table/list.
- [ ] Skeleton/loading preserves final layout footprint.
- [ ] Local and global error states are implemented and use shared components.
